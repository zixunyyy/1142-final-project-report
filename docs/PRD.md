# 產品需求文件 (PRD)

**專案名稱**：EmpathEye: AI 社交共鳴與情緒分析系統
**專案描述**：本計畫旨在填補數位通訊中的「情感斷層」，提供即時純文字情緒分析與社交回應建議，減少社交誤解並協助使用者掌握溝通氛圍。

## 1. 專案概述
- **背景與動機**：隨著數位通訊普及，純文字互動缺乏語氣與表情，容易導致解讀偏差與社交誤解（研究顯示比例超過 50%）。本計畫旨在填補此「情感斷層」，不僅協助社交障礙者，也具備企業管理與遠距醫療的市場潛力。
- **目標用戶**：
  1. 大學生、高中生
  2. 有情緒紀錄或自我反思需求者
  3. 遠距工作團隊（減少溝通誤解）
  4. 社交焦慮或亞斯伯格症族群（輔助解讀情緒）
  5. 線上客服或心理諮詢專業人員
- **核心價值主張**：透過自然語言處理技術，讓純文字溝通不再冰冷與充滿誤解，並賦予使用者更精準的社交應對能力與情緒覺察力。

## 2. 功能需求

- **功能一：文字情緒分析**
  - 作為一名使用者，我希望輸入一段純文字後能看到其情緒類別與強度，以便了解該段文字的潛在情緒色彩，避免過度解讀。
- **功能二：社交回應建議**
  - 作為一名不善社交的使用者，我希望系統能提供適當的破冰或安撫回覆建議，以便在察覺對方情緒不佳時給予得體的回應。
- **功能三：情緒歷史儀表板**
  - 作為一名想要自我反思的使用者，我希望系統能圖表化呈現我過去的情緒分析紀錄，以便追蹤我的情緒波動趨勢。
- **功能四：專屬帳號與加密儲存**
  - 作為一名重視隱私的使用者，我希望我的對話與分析結果是綁定在我的帳號中，並且只有登入才能檢視，以便保護我的個人隱私。
- **功能五：歷史對話回顧**
  - 作為一名心理諮商師/管理者，我希望能夠以條列方式查看過去分析過的具體對話與建議，以便做為輔導或管理時的參考。

## 3. 非功能需求
- **技術限制**：
  - 前端：HTML / CSS / JavaScript (無框架)
  - 後端：Python + Flask
  - 資料庫：SQLite
- **效能考量**：系統呼叫情緒分析服務時需在 3 秒內回傳結果，且網頁需支援響應式設計（RWD）。
- **安全考量**：資料庫內容應只允許授權用戶（登入後）存取，敏感資料避免明碼顯示。

## 4. MVP 範圍

- **Must Have (必須有)**：
  - 文字情緒分析 (辨識情緒與強度)
  - 社交回應建議 (提供 2-3 個回覆選項)
- **Should Have (應該有)**：
  - 情緒歷史儀表板 (視覺化圖表呈現歷史紀錄)
  - 使用者帳號與登入機制
- **Nice to Have (可以有)**：
  - 串接第三方通訊軟體 (如 Line Bot) [此項目已在此專題列為 Out of Scope，故僅為未來展望]
  - 語音輸入轉文字分析
