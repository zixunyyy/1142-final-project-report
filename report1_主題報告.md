## 一、專題概述（Planning）

### 1.1 動機與背景

> 說明為什麼選擇這個主題，這個問題在生活中或產業中真實存在嗎？

隨著數位通訊普及，純文字互動缺乏語氣與表情，容易導致解讀偏差與社交誤解（研究顯示比例超過 50%）。本計畫旨在填補此「情感斷層」，不僅協助社交障礙者，也具備企業管理與遠距醫療的市場潛力。

### 1.2 目標使用者

> 這個系統是為誰設計的？他們有什麼特性或需求？

1. **社交障礙者（如泛自閉症障礙者）**：難以透過文字精準掌握他人情緒，需要輔助工具協助解讀與應對。
2. **一般大眾**：希望改善數位溝通品質、減少文字對話中誤解的人。
3. **企業人資/管理者**：希望透過文字分析了解團隊內部的情緒狀態與溝通氛圍。
4. **遠距醫療/心理諮商師**：作為了解個案情緒變化的輔助參考工具。

### 1.3 專題目標

> 完成後，這個系統能解決什麼問題？請用條列方式列出 3–5 點。

- 目標一：提供即時的純文字情緒分析，精準辨識情緒類別與強度，減少文字溝通中的誤解。
- 目標二：協助使用者辨識他人的情緒狀態後，給予適當的社交共鳴與回應建議。
- 目標三：記錄與追蹤使用者的情緒變化歷程，提供視覺化的情緒圖表，作為自我反思與輔導參考。

## 二、需求分析（Requirements Analysis）

### 2.1 功能性需求（Functional Requirements）

> 系統「必須能做到」的事情。請列出核心功能（至少 3 項）。

| 功能編號 | 功能名稱 | 功能描述 |
| -------- | -------- | -------- |
| F-01     | 文字情緒分析 | 使用者輸入一段文字，系統能分析出對應的情緒標籤（如快樂、生氣、悲傷）與情緒強度。 |
| F-02     | 社交回應建議 | 系統根據分析出的情緒結果，提供 2-3 個適合的情境回覆建議，幫助使用者破冰或化解衝突。 |
| F-03     | 情緒歷史與追蹤 | 儲存使用者的分析紀錄，並提供個人化的情緒歷史儀表板與圓餅圖、折線圖統計。 |

### 2.2 非功能性需求（Non-functional Requirements）

> 系統的品質要求，例如效能、安全性、易用性等。

| 類別   | 需求描述 |
| ------ | -------- |
| 效能   | 系統呼叫情緒分析 API 或模型時，必須在 3 秒內回傳分析結果與建議。 |
| 安全性 | 使用者的分析紀錄與歷史對話內容需進行基本保護，僅限登入的本人帳號查看。 |
| 易用性 | 介面設計需簡潔直覺（支援 RWD），輸入框與分析結果呈現需清晰，降低使用門檻。 |

### 2.3 範圍限制（Out of Scope）

> 明確說明這次專題「不會」做的功能，避免範圍膨脹。

- 本次不包含：語音情緒分析、臉部表情影像辨識。
- 本次不包含：串接至第三方即時通訊軟體（如 LINE、Discord、Messenger）成為自動回覆機器人，僅作為獨立的網頁應用系統。
